Ansible Role for Managing your Minecraft Server¶

This role installs Minecraft or Spigot and configures it to run under systemd or Supervisor. Its recomendet to use the systemd process managment.

For all steps (Development,starting the Server and executing tests) the User must accept the Minecraft EULA, by own configured property!

Role scope Description¶

  • initial installation
  • handle mc server version updates
  • handle plugins

Out of Scope¶

  • executing backups and recovery
  • healthy checks like Minecraft-Region-Fixer
  • handle utility services like filebeat or prometheus
  • install additional Tools like rcon-cli.

All of this is needet but not a part of this role!

A finished installation can be looks like:

[xxxx@minecraft /]$ tree -d -L 4 /opt/
/opt/
└── minecraft
    └── server
        ├── current -> /opt/minecraft/server/releases/1.13.1
